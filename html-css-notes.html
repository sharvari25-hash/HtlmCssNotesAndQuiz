1) HTML Start: Tags, Elements, Block vs Inline

Definition: HTML (HyperText Markup Language) is used to create the structure of web pages.

HTML = Structure CSS = Design JavaScript = Behavior
Tag Structure and Types

<tag>content</tag> is a normal paired tag. Example: <p>Hello</p>.

Void/self-closing tags do not have closing tags: <br>, <hr>, <img>, <input>.

Memory Trick: Block = Big Box, Inline = In the Line.
Block vs Inline Visual Demo
I am display:block (new line, full width behavior).
I am inline I stay in same line inline-block
TypeMain BehaviorCommon Tags

InlineStays in same line, content width onlyspan a b i em strong sub sup
Formatting Tags (Text Decoration Tags)

b (visual bold), strong (important meaning), i (visual italic), em (emphasis), u, ins, del, mark, H2O (subscript), 102 (superscript).

Best practice: Use strong/em for meaning, not only for style.

8) Remaining HTML Topics (Definitions)

Global Attributes

  • id: unique element identifier.
  • class: reusable style/script selector.
  • style: inline CSS style.
  • title: tooltip text on hover.
  • lang: language metadata.
  • dir: text direction (ltr/rtl).
  • hidden: hide element.
  • tabindex: keyboard tab order.
  • contenteditable: editable content region.

  • alt on images for screen readers.
  • label for linked with input id.

Use semantic HTML first, then ARIA only where necessary.

12) CSS Box Model (Everything Related)

Definition: Every element is a rectangular box: content -> padding -> border -> margin.

Content
Core Box Model Properties
PartProperty GroupDefinition
Contentwidth, height, min/max-width, min/max-heightActual content area dimensions.
Paddingpadding, padding-top/right/bottom/leftInner space between content and border.
Borderborder familyEdge line around padding/content.
Marginmargin, margin-top/right/bottom/leftOuter space around the element.
Sizing modebox-sizingDefines how width/height are calculated.
Shorthand Rules + TRBL Memory 
/* One value -> all sides */
margin: 10px;

/* Two values -> vertical | horizontal */
padding: 12px 20px;

/* Three values -> top | horizontal | bottom */
margin: 10px 18px 22px;

/* Four values -> top right bottom left (TRBL) */
padding: 10px 20px 30px 40px;
Memory Trick: TRBL = Top Right Bottom Left.
Box Sizing and Width Calculation 
/* Default */
box-sizing: content-box;
/* width applies to content only */

/* Recommended */
* { box-sizing: border-box; }
/* width includes padding + border */

Example calculation: If width = 200px, padding left+right = 40px, border left+right = 10px

  • content-box total = 250px
  • border-box total = 200px
Margin Collapsing + Centering
  • Vertical margins of adjacent block elements can collapse into one margin (largest value wins).
  • Horizontal margins do not collapse.
  • Center a fixed-width block with margin: 0 auto;.
  • Negative margins are allowed and shift elements.

13) Layout Core: Float, Overflow, Display, Position, Z-Index

Float + Clear (Legacy but Important)

Definition: float moves an element left/right; surrounding content wraps around it. clear stops wrapping from floated elements.

float

Float was originally made for text wrapping around images. The floated box is removed from normal flow, so parent height can collapse if not cleared.

clear: both; clears left and right floats.
.image { float: left; margin-right: 12px; }
.clearfix::after { content: ""; display: block; clear: both; }
Overflow Property (Content Exceeding Box)

Definition: Controls what happens when content is bigger than its container.

visible
This is long long long long content that can spill outside container.
hidden
This is long long long long content that gets clipped and hidden.
auto
This is long long long long content that shows scrollbars only when needed. 
overflow: visible | hidden | scroll | auto;
overflow-x: hidden;
overflow-y: auto;

Media Query Definitions
QueryDefinition

}
Responsive Rules Checklist
  • Always include <meta name="viewport" content="width=device-width, initial-scale=1.0" />.
  • Use fluid widths and avoid fixed large pixel widths for layout containers.
  • Use max-width: 100%; for images/video to prevent overflow.
  • Prefer mobile-first breakpoints.
  • Use spacing and font scales that adapt using rem and clamp().
